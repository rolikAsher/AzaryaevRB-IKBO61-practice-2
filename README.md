## Минимальный CLI-прототип

### Цель проекта
Разработать инструмент визуализации графа зависимостей для менеджера пакетов.  
На этапе 1 создаётся минимальное CLI-приложение, которое принимает пользовательские параметры через командную строку и проверяет их корректность.

### Функциональные возможности
1. Получение пользовательских параметров через аргументы командной строки:
   - `--package` (`-p`) — имя анализируемого пакета  
   - `--repo` (`-r`) — путь к тестовому репозиторию или URL-адрес  
   - `--test_mode` (`-t`) — режим работы (`readonly`, `write`, `simulate`)  
   - `--filter` (`-f`) — подстрока для фильтрации пакетов  
2. Проверка корректности введённых параметров.  
3. Вывод всех параметров в формате ключ=значение.  
4. Обработка ошибок и информативные сообщения пользователю.  

### Пример использования
python dep_viz_cli.py -p mypkg -r ./test_repo -t readonly -f util

Результат:
Запущено приложение с параметрами:
package=mypkg
repo=C:\Users\Ашер\PycharmProjects\PythonProject2\test_repo
test_mode=readonly
filter=util

### Обработка ошибок
Программа проверяет:
- наличие имени пакета;
- существование репозитория;
- допустимые значения режима;
- тип аргументов.

В случае ошибки пользователь получает понятное сообщение.
